<task type="auto">
  <name>Task 1: Integrate Term_Linker into Content_Filter popovers</name>
  <files>includes/class-content-filter.php</files>
  <action>
Modify the `create_popover()` method to process short descriptions through Term_Linker.

**Location:** Line 327-329 in `create_popover()` method

**Current code:**
```php
if ( ! empty( $entry['short_description'] ) ) {
    $popover_html .= sprintf( '<p>%s</p>', esc_html( $entry['short_description'] ) );
}
```

**Replace with:**
```php
if ( ! empty( $entry['short_description'] ) ) {
    // Link glossary terms within the short description, excluding self.
    $short_desc = Term_Linker::link_terms_in_text(
        $entry['short_description'],
        $entry['id']
    );
    $popover_html .= sprintf( '<p>%s</p>', wp_kses_post( $short_desc ) );
}
```

**Important changes:**
1. Call `Term_Linker::link_terms_in_text()` with `$entry['id']` to prevent self-linking
2. Change `esc_html()` to `wp_kses_post()` because the text now contains `<a>` tags
   - This is safe because the link HTML is generated by our code, not user input
   - `wp_kses_post()` allows standard post HTML including anchor tags

**No other changes needed** - the method signature and overall structure remain the same.
