<task type="auto">
  <name>Task 1: Create Term_Linker utility class</name>
  <files>includes/class-term-linker.php</files>
  <action>
Create a new PHP class `PP_Glossary\Term_Linker` with these specifications:

1. **Namespace and security header** - Match existing class files (namespace PP_Glossary, WPINC check)

2. **Static cache property** - `private static $entries_cache = null` for request-level caching

3. **Main method `link_terms_in_text(string $text, int $exclude_id = 0): string`**:
   - Return early if `$text` is empty or `Settings::get_glossary_page_url()` is empty
   - Get entries via `get_entries()` (cached)
   - For each entry (skip if `$entry['id'] === $exclude_id`):
     - Call `replace_first_term_occurrence()` for this entry
   - Return modified text

4. **Private method `get_entries(): array`**:
   - Return from cache if `$entries_cache !== null`
   - Query `pp_glossary` posts (all published, no limit)
   - For each post:
     - Get data via `Meta_Boxes::get_entry_data($post_id)`
     - Skip entries with `disable_autolink` enabled
     - Build entry array with: id, slug, title, terms (title + non-empty synonyms), case_sensitive
   - Sort entries by longest term first (exactly like Content_Filter does)
   - Store in cache and return

5. **Private method `replace_first_term_occurrence(string $text, array $entry, string $glossary_url): string`**:
   - For each term in `$entry['terms']`:
     - Build regex pattern: `/\b({term})\b(?![^<]*>)/` with `u` or `iu` flags based on case_sensitive
     - Use `preg_match()` with `PREG_OFFSET_CAPTURE` to find first match
     - If match found:
       - Create link HTML: `<a href="{glossary_url}#{slug}" class="pp-glossary-link">{matched_term}</a>`
       - Use `substr_replace()` at the offset to replace only that occurrence
       - Return modified text immediately (only first occurrence)
   - Return original text if no match

**Pattern references:**
- Regex pattern from Content_Filter line 232: `/\b(' . preg_quote( $term, '/' ) . ')\b(?![^<]*>)/`
- Sorting logic from Content_Filter lines 170-177
- Entry data structure from Content_Filter lines 156-164

**Important:**
- Use `esc_url()` for href, `esc_attr()` for slug, `esc_html()` for term text
- Class will be autoloaded by existing autoloader (no changes to pp-glossary.php needed)
