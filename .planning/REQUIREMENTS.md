# Requirements: Glossary by Progress Planner

**Defined:** 2026-02-04
**Core Value:** Glossary terms mentioned anywhere should become discoverable learning opportunities

## v1 Requirements

Requirements for the nested term linking feature (GitHub Issue #35).

### Nested Term Linking

- [ ] **NEST-01**: Terms in glossary long descriptions are auto-linked to glossary page anchors
- [ ] **NEST-02**: Terms in glossary short descriptions are auto-linked to glossary page anchors
- [ ] **NEST-03**: A term does not link to itself in its own definition
- [ ] **NEST-04**: Nested links work in Glossary List block output
- [ ] **NEST-05**: Nested links work in popover content
- [ ] **NEST-06**: Nested links use same dotted underline style as inline term triggers
- [ ] **NEST-07**: Synonyms are matched in addition to primary term titles

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Nested popovers | UX complexity, recursion risk, decided against in planning |
| Setting to enable/disable | Keep simple, feature is universally useful |
| Depth limiting controls | One level only, no need for configuration |
| Different behavior per context | Same linking everywhere for consistency |
